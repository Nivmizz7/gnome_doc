GNOME 0.1
Date de sortie : août 1998

GNOME 0.1, la première version publique du projet GNOME, a été lancée en août 1998. Elle marque un tournant important dans l’histoire de GNOME, car elle a permis de présenter un environnement de bureau libre et open-source aux utilisateurs de systèmes Unix et Linux. Bien que GNOME 0.1 ait encore été dans une phase expérimentale, elle a jeté les bases pour ce qui allait devenir un environnement de bureau mature et populaire.

Caractéristiques de GNOME 0.1
Objectif Principal
GNOME 0.1 visait à fournir une alternative libre et open-source aux environnements de bureau propriétaires comme CDE (Common Desktop Environment) et à offrir une plateforme unifiée pour les applications de bureau sur les systèmes GNU/Linux et Unix. Bien que rudimentaire, cette version avait pour objectif de poser les bases de GNOME en tant qu’environnement de bureau complet.

Technologies
GTK+ : GNOME 0.1 a été construit sur la bibliothèque GTK+, qui permet de créer des interfaces graphiques dans les applications. GTK+ était une alternative libre et open-source à des bibliothèques comme Motif, utilisées dans des environnements propriétaires.
Langage C : GNOME 0.1 était principalement écrit en langage C, utilisant un modèle objet basé sur le GObject qui allait être la pierre angulaire du développement futur de GNOME.
Interface Utilisateur
L'interface de GNOME 0.1 était encore très basique. Il n'y avait pas encore de gestionnaire de fenêtres avancé comme ceux des versions suivantes, et l'interface était assez minimaliste. Il n'existait pas de véritables composants d'interface unifiés, avec des éléments de base comme des fenêtres et des menus, mais l'ergonomie n'était pas encore optimale pour l'utilisateur moyen.

Pas de gestion de sessions
GNOME 0.1 ne disposait pas encore de la gestion de sessions comme dans les versions modernes de GNOME. Cela signifiait qu'il fallait lancer manuellement les applications à chaque démarrage du bureau. Le concept de session, permettant de sauvegarder l'état des applications ouvertes, n'était pas encore implémenté.

Composants Restreints
Les outils et applications inclus dans GNOME 0.1 étaient limités. Il n’y avait pas encore de gestionnaire de fichiers complet comme Nautilus, qui serait ajouté plus tard dans GNOME. Les utilisateurs pouvaient se contenter d’un ensemble d’outils rudimentaires, comme un éditeur de texte et des utilitaires de base pour commencer à interagir avec le système.

Support des applications
L'écosystème GNOME était encore embryonnaire à ce stade. Aucune application majeure n'était encore développée spécifiquement pour GNOME, et le système était en grande partie orienté vers les développeurs qui créaient des applications utilisant la bibliothèque GTK+.

Compatibilité avec les distributions
Bien que GNOME 0.1 était une version expérimentale, il a été conçu pour être compatible avec les distributions GNU/Linux ainsi que d'autres systèmes Unix-like. Voici les principaux points concernant la compatibilité à cette époque :

1. Compatibilité avec GNU/Linux
Red Hat et Debian ont été parmi les premières distributions à s'intéresser à GNOME 0.1. Cependant, GNOME n’était pas encore intégré par défaut dans ces distributions à ce stade, et les utilisateurs devaient souvent compiler le projet à partir du code source.
Slackware, une des premières distributions, a également commencé à proposer un support pour GNOME 0.1, bien que ce support n'était pas officiel et que des ajustements manuels étaient nécessaires.
2. Unix et autres systèmes compatibles
Portabilité : GNOME 0.1 était conçu pour être portable sur différents systèmes Unix, mais la compatibilité et la stabilité étaient souvent limitées. Sur des systèmes comme Solaris, BSD ou AIX, GNOME 0.1 pouvait être installé, mais les performances étaient inégales.
GTK+ offrait la possibilité de développer des applications multiplateformes, mais à ce stade, l'installation et la configuration sur des systèmes autres que Linux étaient complexes et nécessitaient souvent des ajustements spécifiques.
3. Distribution GNOME Spécifique
À l’époque, il n’existait pas encore de distribution spécifiquement dédiée à GNOME. Les utilisateurs pouvaient installer GNOME aux côtés d'autres environnements de bureau comme KDE ou FVWM.
4. Problèmes de dépendances et de stabilité
Dépendances : L'une des difficultés majeures pour les utilisateurs était la gestion des dépendances. Les bibliothèques nécessaires à GNOME 0.1, comme GTK+, n'étaient pas toujours préinstallées, et l’installation nécessitait des connaissances avancées des systèmes.
Stabilité : GNOME 0.1 était encore un logiciel expérimental et avait une stabilité limitée. Des plantages et des comportements instables étaient fréquents, ce qui rendait son utilisation difficile pour les utilisateurs non techniques.

Installation de GNOME 0.1
  Dépendances nécessaires :
    1. GCC (GNU Compiler Collection)
      Vérification :
          gcc --version
      Installation si nécessaire :
        Ubuntu/Debian : 
          sudo apt-get install build-essential
        Fedora : 
          sudo dnf install gcc
        Arch Linux : 
          sudo pacman -S gcc
          
    2. GTK+
      Vérification :
          gtk-config --version
      Installation si nécessaire :
        Ubuntu/Debian : 
          sudo apt-get install libgtk2.0-dev
        Fedora : 
          sudo dnf install gtk2-devel
        Arch Linux : 
          sudo pacman -S gtk2

    3. Make
      Vérification :
          make --version
      Installation si nécessaire :
        Ubuntu/Debian : 
          sudo apt-get install make
        Fedora : 
          sudo dnf install make
        Arch Linux : 
          sudo pacman -S make

    4. Autoconf
      Vérification :
          autoconf --version
      Installation si nécessaire :
        Ubuntu/Debian : 
          sudo apt-get install autoconf
        Fedora : 
          sudo dnf install autoconf
        Arch Linux : 
          sudo pacman -S autoconf

    5. Automake
      Vérification :
          automake --version
      Installation si nécessaire :
        Ubuntu/Debian : 
          sudo apt-get install automake
        Fedora : 
          sudo dnf install automake
        Arch Linux : 
          sudo pacman -S automake

    6. Libtool
      Vérification :
          libtool --version
      Installation si nécessaire :
        Ubuntu/Debian : 
          sudo apt-get install libtool
        Fedora : 
          sudo dnf install libtool
        Arch Linux : 
          sudo pacman -S libtool

Étapes d'installation :

  Télécharger les sources :
    wget https://download.gnome.org/desktop/0.x/gnome-0.1.tar.gz

  Extraire l'archive :
    tar -xvzf gnome-0.1.tar.gz

  Se déplacer dans le répertoire extrait :
    cd gnome-0.1

  Lancer le script de configuration :
    ./configure

  Compiler le code source :
    make

  Installer GNOME :
    sudo make install

Et voilà ! Redémarre ton système, et GNOME 0.1 sera installé :).

GNOME 0.0

Date de sortie : août 1997

GNOME 0.0, la toute première version de GNOME (GNU Network Object Model Environment), lancée en août 1997, marque le début d’un projet ambitieux visant à créer un environnement de bureau libre et convivial, principalement destiné aux systèmes Unix et Linux. Cette version était principalement un point de départ et ne ressemblait pas à ce que GNOME est devenu aujourd’hui. En fait, GNOME 0.0 était encore un ensemble de projets et d’idées sans une structure définie.

Caractéristiques de GNOME 0.0 :
Objectif Principal : GNOME a été lancé pour fournir une alternative libre et open-source aux bureaux propriétaires comme CDE (Common Desktop Environment) et les solutions GNOME (Graphical Network Operating System) d’alors. Il visait à intégrer un environnement de bureau facilement utilisable pour les utilisateurs de GNU/Linux.
Technologies : Cette version a mis l’accent sur le modèle objet basé sur le langage C et la bibliothèque GTK+ (GIMP Toolkit). GTK+ a été choisi comme fondation, car il offrait une alternative à la bibliothèque Motif, payante et non libre.
Interface Utilisateur : L’interface était assez basique, comparée à ce qui allait suivre. Aucune véritable interface utilisateur intégrée, avec un gestionnaire de fenêtres rudimentaire, qui ne permettait pas une expérience véritablement cohérente.
Pas de gestion de sessions : GNOME 0.0 n’avait pas encore de gestion de sessions, ce qui signifie qu’il était nécessaire de lancer manuellement les applications à chaque démarrage du bureau. Le concept de sauvegarde d’une session n’était pas encore en place.
Composants Restreints : Les outils étaient très limités, et GNOME 0.0 n’incluait pas de gestionnaire de fichiers complet comme Nautilus (qui allait arriver plus tard). À ce stade, GNOME se composait de simples utilitaires et applications basiques, dans une structure encore expérimentale.
Support des applications : Il n'y avait pas d'écosystème d'applications construit autour de GNOME à cette époque. La priorité était avant tout de définir une fondation sur laquelle les autres applications pouvaient se développer.
En résumé, GNOME 0.0 était une version très embryonnaire, un début de projet dont l’objectif était de poser les bases d’un environnement de bureau libre, mais il manquait encore d'une structure solide, d'applications utilisables au quotidien et d'une véritable expérience utilisateur. Il fallait attendre les versions suivantes pour voir émerger des améliorations notables et des fonctionnalités plus robustes.

Compatibilité avec les distributions
Bien que GNOME 0.0 soit une version très précoce et encore peu développée de l'environnement de bureau, il est important de noter que, dès ses premières étapes, il a été conçu pour être compatible avec des distributions GNU/Linux ainsi que d'autres systèmes Unix-like. Cependant, en raison de son état encore expérimental et limité, la compatibilité avec les distributions était restreinte et souvent plus théorique que réelle à ce stade. Voici quelques détails sur les distributions compatibles à cette époque :

1. Compatibilité avec GNU/Linux :
Les premières distributions de GNOME étaient principalement axées sur les systèmes GNU/Linux, car c'était l'écosystème dans lequel GNOME était censé évoluer. Toutefois, en 1997, les distributions GNU/Linux étaient encore dans une phase de développement où la gestion des environnements de bureau n'était pas aussi mature que celle d'aujourd'hui. Par conséquent, GNOME 0.0 était rarement intégré par défaut dans les distributions, mais les utilisateurs pouvaient compiler le projet à partir du code source.
Des distributions comme Red Hat et Debian étaient parmi les premières à s'intéresser à GNOME, car elles étaient plus orientées vers la communauté de développeurs. Par contre, la version GNOME 0.0 n'était pas préinstallée dans ces distributions, mais les utilisateurs pouvaient la télécharger et l'installer manuellement.
Slackware, étant une des distributions les plus anciennes, était également compatible avec GNOME dès ses premières versions, mais le support n'était pas officiel et nécessitait des ajustements manuels pour l'intégration complète dans l'environnement de bureau.

2. Unix et autres systèmes compatibles :
GNOME 0.0 visait également à être portable sur d'autres systèmes Unix, bien qu'il ne fût pas aussi fluide sur des plateformes comme Solaris, BSD, ou AIX. La bibliothèque GTK+ utilisée par GNOME était conçue pour être portable, mais au début, les performances et la stabilité sur ces systèmes n'étaient pas toujours garanties.
Le projet GNOME a, dès ses débuts, cherché à offrir une compatibilité multiplateforme, bien que cela se soit développé lentement au fil des années. Sur d'autres systèmes Unix, l’installation de GNOME 0.0 pouvait nécessiter une configuration complexe et des ajustements spécifiques à chaque système. L'absence de gestion de sessions et la simplicité de l'interface rendaient son utilisation plus adaptée à des utilisateurs techniques et expérimentés, principalement sur des systèmes GNU/Linux.

3. Distribution GNOME Spécifique :
À ce moment-là, il n'existait pas de distribution spécialement dédiée à GNOME, et l'idée d'une distribution "GNOME-based" n'était pas encore en vigueur. Les utilisateurs qui voulaient essayer GNOME devaient l'installer dans leur environnement Linux existant, souvent aux côtés d'autres environnements de bureau comme KDE ou FVWM. Les distributions qui prenaient en charge GNOME 0.0 proposaient souvent une option pour ajouter GNOME après l’installation, mais ce n'était pas une priorité.

4. Problèmes de dépendances et de stabilité :
Comme avec toute version précoce d'un logiciel, l'un des principaux défis pour les utilisateurs de GNOME 0.0 était la gestion des dépendances. Les bibliothèques nécessaires à GNOME 0.0, comme GTK+, n'étaient pas toujours préinstallées dans les distributions de l'époque. Cela signifiait que les utilisateurs devaient avoir une bonne connaissance des systèmes pour installer et configurer ces dépendances.
La stabilité était également un problème majeur. GNOME 0.0 était encore en phase de développement, et le code était loin d'être stable ou optimisé pour une utilisation quotidienne. Les erreurs fréquentes, les plantages et la lenteur étaient des aspects communs du travail avec GNOME 0.0, ce qui décourageait beaucoup d'utilisateurs non techniques. La majorité des utilisateurs préféraient attendre que des versions plus matures de GNOME soient publiées avant de les adopter pleinement.

5. Les utilisateurs techniques :
En raison de l'immaturité de GNOME 0.0, seuls les utilisateurs techniques, comme les développeurs et les contributeurs à la communauté, étaient en mesure de l'installer et de l'utiliser dans des environnements de production. Les autres utilisateurs préféraient se tourner vers des environnements de bureau plus robustes, comme FVWM ou CDE (Common Desktop Environment), qui étaient plus stables et largement utilisés dans les distributions GNU/Linux de l’époque.

Conclusion sur la compatibilité des distributions :
GNOME 0.0 était compatible avec les distributions GNU/Linux telles que Red Hat, Debian, Slackware, mais la compatibilité était encore très expérimentale. L'absence de packages précompilés et de configurations automatiques rendait l'installation difficile, et sa stabilité limitée le rendait difficile à utiliser sur des systèmes de production. Le support était principalement offert par la communauté de développeurs de GNOME, et ce n'est qu'avec les versions suivantes que l'environnement de bureau serait intégré de manière plus fluide dans les distributions GNU/Linux.

Installation : 
  Dépendances : 
1. GCC (GNU Compiler Collection)

  Vérification :
    gcc --version

Si GCC est installé, cette commande renverra la version du compilateur.

Installation si nécessaire :
  Sur Ubuntu/Debian :
    sudo apt-get install build-essential
  Sur Fedora :
    sudo dnf install gcc
  Sur Arch Linux :
    sudo pacman -S gcc

2. GTK+

  Vérification :
    gtk-config --version

Si GTK+ est installé, cette commande renverra la version de GTK+.

Installation si nécessaire :
  Sur Ubuntu/Debian :
    sudo apt-get install libgtk2.0-dev
  Sur Fedora :
    sudo dnf install gtk2-devel
  Sur Arch Linux :
    sudo pacman -S gtk2

3. Make

  Vérification :
    make --version

Si make est installé, cette commande renverra la version de make.

Installation si nécessaire :
  Sur Ubuntu/Debian :
    sudo apt-get install make
  Sur Fedora :
    sudo dnf install make
  Sur Arch Linux :
    sudo pacman -S make

4. Autoconf

  Vérification :
    autoconf --version

Si Autoconf est installé, cette commande renverra la version de Autoconf.

Installation si nécessaire :
  Sur Ubuntu/Debian :
    sudo apt-get install autoconf
  Sur Fedora :
    sudo dnf install autoconf
  Sur Arch Linux :
    sudo pacman -S autoconf

5. Automake

  Vérification :
    automake --version

Si Automake est installé, cette commande renverra la version de Automake.

Installation si nécessaire :
  Sur Ubuntu/Debian :
    sudo apt-get install automake
  Sur Fedora :
    sudo dnf install automake
  Sur Arch Linux :
    sudo pacman -S automake

6. Libtool

  Vérification :
    libtool --version

Si Libtool est installé, cette commande renverra la version de Libtool.

  Installation si nécessaire :
    Sur Ubuntu/Debian :
      sudo apt-get install libtool
    Sur Fedora :
      sudo dnf install libtool
    Sur Arch Linux :
      sudo pacman -S libtool

Une fois toutes les dépendances installées, on peut passer à l'installtion de GNOME 0.0.

Commandes pour installer GNOME 0.0  :

  Télécharger les sources :
    wget https://download.gnome.org/desktop/0.x/gnome-0.0.tar.gz
  Extraire l'archive :
    tar -xvzf gnome-0.0.tar.gz
  Se déplacer dans le répertoire extrait :
    cd gnome-0.0
  Lancer le script de configuration :
    ./configure
  Compiler le code source :
    make
  Installer GNOME :
    sudo make install

Et voila, redémarrez et votre machine aura GNOME 0.0 installé. :)
